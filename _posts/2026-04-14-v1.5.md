I'm thrilled to officially announce **Sniffnet v1.5**!<br><br>
The release includes a new major functionality desired by many users for a long time:
Sniffnet is now able to **show the applications and programs that are using your network bandwidth**!

<div align="center">
<img alt="" class="paddedGalleryImg" title="Programs" src="{{ 'assets/img/post/v1.5/programs.png' | relative_url }}">
</div>

This is a huge step forward for Sniffnet, allowing you to closely monitor suspicious executables running undetected and to keep an eye on the network usage of the applications you use the most.

As you can see in the video below, Sniffnet now shows all programs actively using the network, and for each of them it's possible to inspect the list of opened network connections.<br>
Each program can also be saved as **favorite** to be easily identified in the future and to get **notifications** whenever it generates network traffic.

<div align="center">
<img alt="" class="paddedGalleryImg" title="Notifications page" width="100%" src="{{ 'assets/img/post/v1.5/cover.png' | relative_url }}">
</div>

You've probably already read about the struggles and challenges I faced while implementing this feature in the <a href="{{ 'news/process-identification' | relative_url }}">previous blog post</a>,
so it's a huge pleasure and excitement for me to finally share that this is now available for everyone to use on Windows, Linux, macOS, and even BSD-based operating systems.

<hr>

But there is more!

In addition to program identification, Sniffnet v1.5 also lets you import **custom IP blacklists**
to be warned about potentially malicious connections.

Moreover, the initial page of the app now shows **traffic previews for all the network adapters** on your machine,
allowing you to quickly identify active interfaces and to get an overview of their traffic without having to start an analysis session for each of them individually.
